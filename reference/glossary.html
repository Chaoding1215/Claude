Cybersecurity Glossary

Canonical vocabulary for presales conversations. Terms are organized by domain. When a term appears in a lesson, the lesson number is noted. Print this for desk reference.

Foundations Threat Actors Attack Techniques Endpoint Network Cloud Identity AppSec SOC / SIEM Zero Trust Advanced Topics

Foundations

The universal vocabulary every security conversation assumes.

L1
CIA Triad

The three core security properties: Confidentiality (only authorized parties can read data), Integrity (data has not been tampered with), Availability (systems are accessible when needed). Every security control protects one or more of these.

Avoid: "the three pillars" — use "the CIA triad" or name the properties explicitly.
L1
Confidentiality

Information is accessible only to those authorized to see it. Protected by encryption, access controls, and DLP.

L1
Integrity

Information is accurate and has not been altered by unauthorized parties. Protected by hashing, digital signatures, and audit logs.

L1
Availability

Systems and data are accessible when needed by authorized users. Protected by redundancy, DDoS mitigation, and incident response.

L1
Non-repudiation

An action can be definitively attributed to a party who cannot deny it. Achieved through digital signatures and audit trails. Often bundled under Integrity.

L1
Threat model

A structured analysis of which adversaries are relevant, what assets they would target, and how they might attack. Security is always relative to a threat model — there is no absolute security.

Avoid: saying something "is secure" without naming the threat model.
L1
Defense in depth

Layered security controls so that no single failure leaves the organization unprotected. If one layer is bypassed, another catches the attacker. The dominant defensive strategy.

Avoid: "security in depth" — the correct term is "defense in depth."
L1
Defender's dilemma

The asymmetry where attackers need to succeed once, while defenders must succeed every time. Explains why detection and response matter as much as prevention.

L1
Attack surface

The total set of points where an attacker could attempt to enter or extract data from a system. Reducing attack surface is a primary security objective.

Vulnerability

A weakness in a system that could be exploited by an attacker. Vulnerabilities are assigned CVE identifiers and CVSS severity scores.

Exploit

Code or a technique that takes advantage of a vulnerability to cause unintended behavior in a system.

CVE Common Vulnerabilities and Exposures

A standardized identifier for publicly known security vulnerabilities (e.g., CVE-2021-44228 = Log4Shell). Published by MITRE, maintained in the NVD.

CVSS Common Vulnerability Scoring System

A 0–10 numerical score measuring the severity of a vulnerability. 9.0–10.0 = Critical. Used to prioritize patching.

L1
Patch

A software update that fixes a vulnerability. The time between patch release and deployment is a major risk window — most exploits target known, unpatched vulnerabilities.

Threat Actors

Who attacks, and why.

L2
Nation-state actor

A government-sponsored threat actor with significant resources, custom tooling, and long-term patience. Motivations: espionage, sabotage, influence operations. Examples: APT29 (Russia), APT41 (China), Lazarus Group (North Korea).

L2
APT Advanced Persistent Threat

A sophisticated, long-running attack campaign — typically state-sponsored or organized crime — that remains hidden in a target network for months or years. "Persistent" is the defining characteristic.

Avoid: using "APT" as a synonym for any attack — it specifically implies advanced capability and long dwell time.
L2
Organized cybercrime

Professional criminal groups motivated by financial gain. Operate structured businesses including HR, affiliate programs, and victim support desks. Primary operators of ransomware-as-a-service platforms.

L2
Hacktivist

An attacker motivated by ideology or political goals rather than financial gain. Typical methods: DDoS, website defacement, data leaks. Example: Anonymous.

L2
Insider threat

A threat originating from within the organization — employees, contractors, or partners with legitimate access. Can be malicious (revenge, financial) or negligent (accidental data exposure).

L2
Script kiddie

A low-skill attacker who uses pre-built tools and exploits without understanding them. Opportunistic — targets the easiest unpatched systems. Volume is high; sophistication is low.

L2
Threat actor

Any individual, group, or organization that poses a threat to systems or data. The umbrella term covering all attacker categories.

Attack Techniques & Concepts

How attacks work — vocabulary for describing attacker behavior.

L2
TTPs Tactics, Techniques, and Procedures

The specific behaviors and methods a threat actor uses to carry out attacks. Tactics = goals; Techniques = methods; Procedures = specific implementations. Catalogued in MITRE ATT&CK.

L2
IOC Indicator of Compromise

Forensic artifacts that indicate an attack has occurred: malicious IP addresses, file hashes, suspicious domains, registry keys. Shared via threat intelligence feeds for blocking and detection.

L2
Zero-day

A vulnerability unknown to the vendor and without a patch. Highly valuable — nation-states stockpile them. Named for the zero days of defense available when first exploited.

Avoid: using "zero-day" to describe any new attack — it specifically means the vendor is unaware and no patch exists.
L2
Cyber Kill Chain

Lockheed Martin's 7-stage model of a targeted attack: Reconnaissance → Weaponization → Delivery → Exploitation → Installation → Command & Control → Actions on Objectives. Security products intervene at specific stages.

L2
MITRE ATT&CK

The industry-standard knowledge base of adversary tactics and techniques, organized as a matrix. Every serious SOC and security vendor maps their capabilities to ATT&CK. 14 enterprise tactics, hundreds of techniques.

Phishing

Deceptive communications (usually email) designed to trick recipients into revealing credentials, clicking malicious links, or opening infected attachments. The most common initial access technique.

Spear phishing

Targeted phishing tailored to a specific individual using personal details to increase credibility. Far more effective than generic phishing; used by APTs and organized crime.

Lateral movement

The attacker's progression from an initial foothold to other systems within the network. Micro-segmentation and Zero Trust limit lateral movement by restricting inter-system access.

Ransomware

Malware that encrypts victim files and demands payment for the decryption key. Modern ransomware groups also exfiltrate data and threaten to publish it ("double extortion").

L2, L5
RaaS Ransomware-as-a-Service

A criminal business model where ransomware developers license their tools to affiliates on a revenue-share basis. Lowers the skill barrier dramatically — affiliates deploy, developers take 20–30% of ransoms.

DDoS Distributed Denial of Service

Flooding a system with traffic from many sources simultaneously to exhaust resources and make it unavailable. An availability attack. Mitigated by scrubbing centers and anycast networks.

Social engineering

Manipulating people into revealing information or taking actions that compromise security. Exploits psychology rather than technology. Phishing is the most common form.

L5
Deepfake

AI-generated synthetic audio or video that impersonates a real person. Increasingly used in social engineering — notably to impersonate executives in fraudulent wire transfer requests.

Blast radius

The scope of damage that results when an attacker achieves their objective. Least-privilege access and micro-segmentation are designed to minimize blast radius.

Dwell time

The time between an attacker gaining initial access and being detected. Industry average has historically been 200+ days. Reducing dwell time is a primary SOC objective.

Endpoint Security

AV Antivirus

Legacy endpoint protection that detects known malware using signature databases. Ineffective against novel or obfuscated malware. Superseded by EDR for serious enterprise use.

L3
EDR Endpoint Detection and Response

Endpoint security that records and analyzes system behavior, enabling detection of novel attacks (not just known signatures), threat hunting, and rollback of ransomware. The current enterprise standard.

L3
XDR Extended Detection and Response

Extends EDR by correlating telemetry across endpoints, network, cloud, identity, and email into a unified detection and response platform. Reduces siloed visibility across domains.

Malware

Malicious software designed to damage, disrupt, or gain unauthorized access to systems. Includes viruses, worms, trojans, ransomware, spyware, and rootkits.

Threat hunting

Proactive human-led search for hidden threats that have evaded automated detection. Analysts form hypotheses and investigate the environment looking for attacker TTPs. Requires EDR telemetry.

Network Security

L3
Firewall / NGFW Next-Generation Firewall

Filters network traffic by rules. NGFWs add application-layer awareness, SSL inspection, and integrated IPS — going beyond traditional port/protocol rules.

L3
IDS / IPS Intrusion Detection / Prevention System

Inspects network traffic for known attack signatures. IDS alerts; IPS actively blocks. Now typically integrated into NGFWs rather than deployed as standalone appliances.

L3
NDR Network Detection and Response

Uses behavioral analysis (not just signatures) to detect anomalous network traffic — lateral movement, C2 communication, data exfiltration. The network equivalent of EDR.

L3
DNS Security

Blocks connections to malicious domains at the DNS resolution level. Catches C2 traffic early in the Kill Chain — before malware can phone home. Often the first layer of network defense.

L4
Micro-segmentation

Dividing a network into small, isolated zones with individual access policies, so lateral movement between zones is blocked even for authenticated users. Implements "assume breach" at the network layer.

VPN Virtual Private Network

Creates an encrypted tunnel between a remote user and the corporate network, granting broad network-level access. Being replaced by ZTNA in Zero Trust architectures due to over-privileged access.

Avoid: recommending VPN as a Zero Trust solution — it is the legacy approach ZTNA replaces.
C2 Command and Control

The channel through which an attacker remotely controls compromised systems. Malware calls home to C2 infrastructure to receive instructions and exfiltrate data. Stage 6 of the Kill Chain.

Cloud Security

L3
CSPM Cloud Security Posture Management

Continuously monitors cloud environments for misconfigurations — public S3 buckets, open databases, overly permissive IAM roles. The most common cause of cloud breaches is misconfiguration, not sophisticated attacks.

L3
CWPP Cloud Workload Protection Platform

Secures cloud workloads: virtual machines, containers, and serverless functions. Runtime protection against attacks on running workloads.

L3
CASB Cloud Access Security Broker

Sits between users and SaaS applications, enforcing security policies — blocking upload of sensitive data to unauthorized apps, controlling access to shadow IT.

L3
CNAPP Cloud-Native Application Protection Platform

Combines CSPM + CWPP (and often more) into a unified platform for securing cloud-native environments. The consolidation trend in cloud security.

Shared responsibility model

In cloud computing, security is divided between the cloud provider (infrastructure) and the customer (data, access, configuration). Most breaches occur in the customer's responsibility zone — misconfigured settings, not compromised AWS infrastructure.

Identity & Access Management

L3
IAM Identity and Access Management

Manages digital identities and controls what resources each identity can access. The foundation of modern security — "identity is the new perimeter."

L3
MFA Multi-Factor Authentication

Requires two or more proofs of identity: something you know (password), something you have (phone/token), something you are (biometric). Blocks the majority of credential-based attacks.

L3
PAM Privileged Access Management

Controls, monitors, and audits access to high-privilege accounts: admin, root, service accounts. A top priority because privileged accounts are attackers' primary target for lateral movement and persistence.

L3
SSO Single Sign-On

One authentication event grants access to multiple applications. Reduces password sprawl and enables centralized access control — revoking access in one place revokes it everywhere.

Least privilege

Users and systems receive the minimum access permissions needed to perform their function — no more. A core Zero Trust principle. Limits blast radius if an account is compromised.

JIT / JEA Just-in-Time / Just-Enough-Access

Privileges are granted only when needed and for the minimum time required, then automatically revoked. Reduces the window of opportunity for attackers who compromise a privileged account.

Credential stuffing

Automated use of stolen username/password combinations from one breach to attack other services — exploiting password reuse. Mitigated by MFA and password managers.

Application Security

L3
OWASP Top 10

The Open Worldwide Application Security Project's canonical list of the most critical web application vulnerabilities. Updated periodically. Includes SQL injection, broken authentication, XSS, insecure deserialization. The reference standard for AppSec conversations.

L3
SAST Static Application Security Testing

Scans source code for vulnerabilities before the application runs — "shift left" security. Catches issues at the earliest and cheapest point to fix them.

L3
DAST Dynamic Application Security Testing

Tests a running application by simulating real attacks from the outside. Finds issues that only appear at runtime — logic flaws, authentication weaknesses, server misconfigurations.

L3
WAF Web Application Firewall

Sits in front of web applications and filters malicious HTTP requests — SQL injection, XSS, bot traffic. Deployed as a cloud service or appliance. A runtime defense; does not fix the underlying vulnerability.

L3
SCA Software Composition Analysis

Identifies known vulnerabilities in open-source libraries and dependencies used by an application. Critical given that modern apps are 80–90% open-source code by volume.

L5
SBOM Software Bill of Materials

A machine-readable inventory of every component, library, and dependency in a software product. Now required by the US government for federal software vendors. Enables rapid impact assessment when a new CVE is published.

SQL injection

An attacker inserts malicious SQL code into an input field to manipulate the database — reading, modifying, or deleting data. One of the oldest and most common web vulnerabilities. Prevented by parameterized queries.

XSS Cross-Site Scripting

Attacker injects malicious JavaScript into a web page viewed by other users, enabling session hijacking, credential theft, or malicious redirects.

SOC / SIEM / SOAR

L3
SOC Security Operations Center

The team and facility responsible for monitoring, detecting, and responding to security events 24/7. May be internal or outsourced (MSSP). The human side of detection and response.

L3
SIEM Security Information and Event Management

Aggregates and correlates logs from across the environment — endpoints, firewalls, cloud, identity — to detect threats and generate alerts. The brain of the SOC. Key vendors: Splunk, Microsoft Sentinel, IBM QRadar.

L3
SOAR Security Orchestration, Automation and Response

Automates repetitive SOC tasks (enriching alerts, blocking IPs, isolating endpoints) and orchestrates multi-step response playbooks. Addresses analyst alert fatigue by eliminating manual busywork.

L3
MTTD Mean Time to Detect

The average time between an attacker gaining access and the defender detecting them. Industry average historically 200+ days. Every SIEM/SOC vendor competes on reducing this number.

L3
MTTR Mean Time to Respond

The average time between detecting an incident and containing/remediating it. SOAR and automation directly reduce MTTR. Both MTTD and MTTR are the primary SOC metrics in any vendor conversation.

Alert fatigue

Analysts becoming desensitized to alerts due to overwhelming volume and high false positive rates. Leads to missed real threats. A primary driver of AI adoption and SOAR investment in security operations.

L2
Threat intelligence

Curated, contextualized data about current threats: attacker TTPs, active campaigns, IOCs, and vulnerability exploitability. Fed into SIEM rules, firewall blocklists, and analyst enrichment workflows.

IR Incident Response

The structured process for detecting, containing, eradicating, and recovering from a security incident. Most enterprises retain an IR firm on retainer before they need one.

Zero Trust

L4
Zero Trust

A security architecture based on "never trust, always verify." Every access request is authenticated and authorized regardless of network location. Assumes breach rather than perimeter integrity. Coined by John Kindervag at Forrester (2010).

Avoid: treating Zero Trust as a product — it is an architecture. No single product "does" Zero Trust.
L4
ZTNA Zero Trust Network Access

Grants access to specific applications — not the whole network — after verifying identity and device health. The primary replacement for VPN in Zero Trust architectures.

L4
Least privilege

Grant the minimum permissions needed for a task, for the minimum time needed. A core Zero Trust principle. Reduces blast radius when credentials are compromised.

L4
Assume breach

Design as if attackers are already inside. Focus on containment, rapid detection, and minimizing lateral movement — not just preventing initial entry.

L4
Perimeter model

The legacy "castle and moat" approach where the firewall defines a trusted interior. Obsolete in the cloud and remote-work era — attackers routinely enter through phishing, supply chain, and insider access.

SASE Secure Access Service Edge

A Gartner-coined architecture that converges networking (SD-WAN) and security (ZTNA, CASB, FWaaS, SWG) into a cloud-delivered service. Enables Zero Trust for distributed workforces without on-premises appliances.

Device posture

The security health status of a device at a given moment — patch level, EDR installed, disk encryption on. Zero Trust gates access on device posture, not just user identity.

Advanced Topics

L5
Post-quantum cryptography (PQC)

Cryptographic algorithms designed to resist attacks from quantum computers. NIST standardized ML-KEM and ML-DSA in 2024, both based on lattice mathematics. Required because Shor's algorithm breaks RSA and elliptic curve on sufficiently powerful quantum computers.

L5
Harvest now, decrypt later (HNDL)

Nation-states collect encrypted communications today and store them, planning to decrypt when quantum computers mature. Makes PQC migration urgent even before quantum computers exist at scale.

L5
Crypto-agility

The ability to swap cryptographic algorithms across a system without major re-architecture. Essential for PQC migration — organizations that built crypto-agile systems can update faster.

L5
Supply chain attack

Compromising a target indirectly by attacking a trusted vendor, software component, or service they rely on. SolarWinds (2020) and XZ Utils (2024) are landmark examples.

L5
Adversarial ML

Crafting inputs specifically designed to fool machine learning models — causing a security ML model to misclassify malicious content as benign. An emerging attack category targeting AI-powered security tools.

L5
SLSA Supply chain Levels for Software Artifacts

A Google-originated framework for hardening software build pipelines against tampering. Defines four levels of supply chain security maturity.

Cyber risk quantification (CRQ)

Expressing cybersecurity risk in financial terms (expected loss, probability of breach) rather than qualitative scores. Enables board-level risk conversations. An active and unresolved research area.

Threat intelligence platform (TIP)

Aggregates, normalizes, and manages threat intelligence from multiple feeds. Enriches SIEM alerts with attacker context and distributes IOCs for blocking across security tools.
