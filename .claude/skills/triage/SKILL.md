Move issues and external pull requests through a state machine of triage roles.

## Roles

**Categories** (pick exactly one per item):
- `bug` — something is broken
- `enhancement` — new feature or improvement

**States** (pick exactly one per item):
- `needs-triage` — not yet evaluated
- `needs-info` — blocked on reporter clarification
- `ready-for-agent` — scoped and self-contained; an agent can implement it
- `ready-for-human` — requires judgment, access, or coordination a human must provide
- `wontfix` — declined; close without action

## Discovery

Prioritize in order:
1. Unlabeled issues and PRs
2. Items labeled `needs-triage`
3. `needs-info` items where the reporter has replied since the last triage note

Surface external PRs (from non-collaborators) separately.

## For each item

1. **Gather context** — read the full issue or PR (including diff for PRs), prior comments, and existing labels. Check the codebase for redundancy (is this already implemented?) and prior rejections.
2. **Recommend** a category and state with brief reasoning.
3. **Verify the claim** — reproduce bugs, confirm PR functionality, or check existing behavior.
4. **Grill if needed** — if requirements are fuzzy and `ready-for-agent` is the target state, run `/grilling` to sharpen them first.
5. **Apply outcomes** — add labels and leave a comment.

## Outcome templates

**needs-info**: Ask the one question that would unblock triage. Template:
> Thanks for the report! To help us investigate, could you share [specific missing info]?
> *(This was generated by AI during triage.)*

**ready-for-agent**: Write an agent brief — a self-contained description of what to build, acceptance criteria, and any blocking dependencies.

**ready-for-human**: Write a human-focused summary explaining why human judgment is required.

**wontfix**: If already implemented, point to where. If declined, explain why without being dismissive.

All AI-generated comments must include: *"This was generated by AI during triage."*

## Notes

- Treat a PR as an issue with attached code — same triage logic applies.
- Preserve prior triage notes; don't re-ask already-resolved questions.
- Triage is only for issues you didn't create. Issues from `/to-issues` are already agent-ready — don't triage them.
