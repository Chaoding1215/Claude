The Security Domains:
The Market Map

The cybersecurity market generates over $200 billion per year and contains thousands of vendors. Without a mental map, it's impossible to navigate. This lesson gives you that map — six domains, each solving a different part of the security problem.

How to use this lesson
When you encounter any security product or vendor — now or in the future — ask: which domain does this live in, what problem does it solve, and where does it sit in the Kill Chain? These three questions will let you place anything.

The Six Domains

🖥️ Endpoint Security Protect

Secures the devices where people work: laptops, desktops, servers, mobile phones. The endpoint is where most attacks ultimately execute — malware runs on a device, ransomware encrypts a device's files, credentials are stolen from a device's memory.

Evolution: AV (1980s) → signature-based detection of known malware → EDR (Endpoint Detection & Response, 2010s) → behavioral detection, threat hunting, rollback → XDR (Extended Detection & Response, 2020s) → correlates endpoint + network + cloud + identity data.

Key problem it solves: Attackers living on endpoints, malware execution, ransomware, credential theft.

Example vendors: CrowdStrike, SentinelOne, Microsoft Defender, Palo Alto Cortex XDR, Trend Micro

🌐 Network Security Protect

Secures the movement of data between systems. In the Kill Chain, network security tools intercept threats at the Delivery stage (blocking malicious traffic before it reaches endpoints) and the C2 stage (detecting compromised machines calling home).

Key technologies:

  • Firewall / NGFW — filters traffic by rules; Next-Gen Firewalls add application awareness and IPS.
  • IDS/IPS — Intrusion Detection/Prevention Systems; inspects traffic for known attack signatures.
  • NDR — Network Detection & Response; uses behavioral analysis to catch threats that bypass signature detection.
  • DDoS Protection — absorbs volumetric attack traffic to maintain availability.
  • DNS Security — blocks connections to malicious domains; catches C2 traffic early.

Key problem it solves: Malicious traffic, C2 communication, lateral movement between systems, DDoS.

Example vendors: Palo Alto Networks, Cisco, Fortinet, Check Point, Zscaler, Cloudflare

☁️ Cloud Security Protect

The cloud introduced a fundamentally new attack surface: misconfigured storage buckets, over-privileged service accounts, and unsecured APIs. Cloud security tools monitor and protect cloud infrastructure, workloads, and data.

Key technologies:

  • CSPM — Cloud Security Posture Management; finds misconfigurations (e.g., public S3 bucket with customer data).
  • CWPP — Cloud Workload Protection Platform; secures VMs, containers, and serverless functions.
  • CASB — Cloud Access Security Broker; enforces policy on SaaS app usage (e.g., blocking upload of sensitive files to personal Dropbox).
  • CNAPP — Cloud-Native Application Protection Platform; combines CSPM + CWPP + more into one platform.

Key problem it solves: Cloud misconfigurations, over-privileged identities, unsecured workloads, shadow IT.

Example vendors: Wiz, Palo Alto Prisma Cloud, CrowdStrike Falcon Cloud, Orca Security, Microsoft Defender for Cloud

🪪 Identity & Access Management (IAM) Protect

"Identity is the new perimeter." When the network perimeter dissolved, controlling who can access what became the primary defense. Over 80% of breaches involve compromised credentials — making identity the most attacked surface today.

Key technologies:

  • IAM — manages user identities, authentication, and authorization at scale.
  • MFA — Multi-Factor Authentication; requires a second proof of identity beyond password.
  • PAM — Privileged Access Management; controls access to high-value accounts (admin, root, service accounts).
  • SSO — Single Sign-On; one login for all apps, reducing password sprawl.
  • Zero Trust — architectural principle: never trust, always verify. Every access request is authenticated and authorized, regardless of network location.

Key problem it solves: Stolen credentials, privilege escalation, lateral movement, insider threats.

Example vendors: Okta, Microsoft Entra ID, CyberArk, BeyondTrust, Ping Identity

🛡️ Application Security (AppSec) Protect

Secures the software organizations build and use. The OWASP Top 10 — a canonical list of the most critical web application vulnerabilities (SQL injection, XSS, broken authentication, etc.) — defines the core threat model.

Key technologies:

  • SAST — Static Application Security Testing; scans source code for vulnerabilities before deployment.
  • DAST — Dynamic Application Security Testing; attacks a running application to find exploitable issues.
  • WAF — Web Application Firewall; sits in front of web apps and blocks malicious HTTP requests.
  • API Security — discovers and protects APIs, which have become the dominant attack surface in modern applications.
  • SCA — Software Composition Analysis; identifies vulnerabilities in open-source dependencies.

Key problem it solves: Vulnerable code shipped to production, injection attacks, API abuse, supply chain vulnerabilities in dependencies.

Example vendors: Veracode, Snyk, Checkmarx, Akamai (WAF), Imperva, Salt Security

🔍 SOC / SIEM / SOAR DetectRespond

The Security Operations Center (SOC) is where defenders monitor, detect, and respond to threats across all other domains. It assumes that some attackers will get through prevention — and focuses on finding them fast and responding before damage is done.

Key technologies:

  • SIEM — Security Information & Event Management; aggregates logs from across the environment, correlates events, and generates alerts. The brain of the SOC.
  • SOAR — Security Orchestration, Automation & Response; automates repetitive SOC tasks and orchestrates response playbooks.
  • Threat Intelligence Platform (TIP) — ingests and manages threat feeds; enriches alerts with attacker context.
  • Threat Hunting — proactive search for hidden attackers who have evaded automated detection.

Key metrics customers care about: MTTD (Mean Time to Detect), MTTR (Mean Time to Respond). Every SOC vendor promises to reduce both.

Key problem it solves: Alert fatigue, slow detection, manual response bottlenecks, lack of visibility across the environment.

Example vendors: Splunk, Microsoft Sentinel, IBM QRadar, Palo Alto XSIAM, Google Chronicle, Elastic Security

The Big Picture: How Domains Connect

No domain works in isolation. The modern security architecture connects all six:

The platform vs. best-of-breed debate
One of the most common customer conversations in security presales. Best-of-breed: pick the top vendor in each domain — maximum capability per category, but harder to integrate. Platform: one vendor covers multiple domains — easier to manage, shared telemetry, but potential capability gaps. Major platforms: Microsoft, Palo Alto, CrowdStrike, Fortinet. Knowing this debate is essential.

Knowledge Check

A customer says: "We found that an attacker had misconfigured one of our AWS S3 buckets as public, exposing 2 million customer records." Which security domain most directly addresses this failure?
A CISO says: "Our MTTD is 200 days — attackers are living in our network for months before we catch them." Which security domain and technology most directly targets this problem?
A developer wants to catch SQL injection vulnerabilities in code before it ships to production. Which AppSec technology applies here?
A prospect says their biggest pain point is stolen employee passwords being used to access company systems from overseas at 3am. Which domain and technology is the primary solution?

Primary source: NIST Cybersecurity Framework 2.0 — The six domains here map onto NIST's Govern / Identify / Protect / Detect / Respond / Recover functions. Understanding both frameworks together gives you the complete picture.
