SASE:
When Networking and Security Merged

SASE (pronounced "sassy") is one of the most important architectural shifts in enterprise IT of the last decade. It's also the category where Palo Alto, Fortinet, and Zscaler fight their most intense competitive battles — and where Huawei has significant ground to cover. Understanding it deeply separates credible presales from buzzword-repeaters.

The Problem SASE Solves

For decades, enterprises ran two separate stacks:

  • Networking — MPLS circuits and SD-WAN routing traffic between offices, data centers, and cloud.
  • Security — Firewalls, proxies, and VPNs, typically sitting at headquarters or data centers.

The model assumed: users are in offices, apps are in data centers. Security appliances at the perimeter inspect traffic before it reaches the internet.

Cloud and remote work destroyed this assumption. When a Tokyo employee opens Salesforce from a café, traffic flows:

Employee laptop → [VPN tunnel to HQ] → HQ firewall → Internet → Salesforce

This "hairpinning" — backhauling all traffic through headquarters — creates latency, bandwidth costs, and a single point of failure. And the security stack at HQ was never designed to inspect encrypted SaaS traffic at scale.

Why this matters competitively
When a customer says "our remote users have terrible performance" or "our MPLS costs are out of control," that's a SASE conversation. Networking pain and security gaps appear together — because they share the same broken architecture.

What SASE Is

Gartner coined SASE (Secure Access Service Edge) in 2019. The definition: converge wide-area networking and security into a single cloud-delivered service, delivered from Points of Presence (PoPs) close to users everywhere.

Instead of routing traffic to a central security stack, SASE inspects it at the network edge — close to the user, wherever they are.

Employee laptop → [SASE PoP — nearest city] → Security inspection → Salesforce

The Five Components of SASE

SASE bundles five capabilities into one cloud service. Knowing each is essential for feature-level conversations:

1. SD-WAN (Software-Defined Wide Area Network)

Replaces expensive MPLS circuits with intelligent routing over any connection (broadband, LTE, MPLS). Routes traffic dynamically based on application type and link quality. The networking half of SASE.

2. ZTNA (Zero Trust Network Access)

Replaces VPN. Users get access only to specific authorized applications — not the whole network — after identity and device verification. Covered in Lesson 4.

3. CASB (Cloud Access Security Broker)

Enforces security policy on SaaS usage: blocks upload of sensitive data to unauthorized apps, discovers shadow IT, controls which cloud services employees can use.

4. FWaaS (Firewall-as-a-Service)

A full next-generation firewall delivered from the cloud — no on-premises appliance. Inspects all traffic including encrypted connections (SSL inspection) at the PoP.

5. SWG (Secure Web Gateway)

Filters web traffic: blocks malicious URLs, enforces acceptable use policy, prevents data exfiltration via web uploads. The replacement for on-premises web proxies.

SSE — SASE's security-only cousin
SSE (Security Service Edge) is SASE minus the SD-WAN — just the security components (ZTNA + CASB + FWaaS + SWG). Organizations that already have SD-WAN, or just want to fix security without changing networking, buy SSE. Zscaler leads here; Palo Alto and Fortinet offer both.

The Competitive Landscape

Zscaler
Pure-cloud SSE leader. Built cloud-native from day one — no legacy appliance code. Strongest in large enterprises transitioning away from VPN. Weakness: no SD-WAN (partners for that).
Palo Alto
Prisma SASE
Strong integrated platform — SD-WAN (via Prisma SD-WAN acquisition) plus best-in-class SSE. Also leads in AI-powered threat detection. Premium pricing. Platform buyer's choice.
Fortinet
FortiSASE
Strong SD-WAN (market leader by units). SASE story is more recent; SSE components less mature than Zscaler/Palo Alto. Strongest with existing Fortinet network customers.
Cisco
Umbrella + Meraki
Strong enterprise relationships, Umbrella for DNS/SWG/CASB, Meraki for SD-WAN. Integration story still maturing post-acquisitions.
Huawei
Strong in SD-WAN (core networking heritage). Security components of SASE (ZTNA, CASB, FWaaS) are significant gaps — noted explicitly in competitive analysis as areas requiring development.

Key Buying Signals in Customer Conversations

  • "Our remote users are complaining about VPN performance" → ZTNA/SASE opportunity
  • "We're spending too much on MPLS" → SD-WAN + SASE conversation
  • "We have no visibility into what SaaS apps our employees are using" → CASB/SASE
  • "We need to secure branch offices without putting appliances in each one" → FWaaS/SASE
  • "We're moving everything to cloud and our perimeter firewall doesn't make sense anymore" → Full SASE transformation

Knowledge Check
